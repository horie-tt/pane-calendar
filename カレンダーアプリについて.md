# 目的
- 日付と曜日だけを確認できる、シンプルなカレンダーが欲しい。
- プロジェクトリーダーがスケジュール作成の際に、工数の算出やMTG日付の確認を行うことを想定している。
- 純粋なカレンダー表示ツール（予定管理は一切なし）。
- 半透明な画面を持ち、ピン留めを行ってあらゆる画面の前面に表示することが可能。
- 工数・営業日の算出も可能
# 機能要件
- 3ヶ月、1ヶ月、1年ごとの表示が出来る。
- 3か月表示の場合は `前月・今月・翌月` を表示させる（年末の場合は来年の月を表示することもある）。今月が中央に表示される。
- 1年表示の場合は今月の月に関係なく、 `1～12月`を表示する。ただし設定ファイル(`config.toml`)にて年度始まりかどうかを設定出来、年度始まりの場合は`4月～(来年の)3月`を表示するものとする。
- 前面に表示できる（ピン留め機能を有しており、有効にすることで常に前面に表示させる）
- 土日祝日は特定の背景色でハイライトされている
- 今日の日付が特定の背景色でハイライトされている
- 日付セルのクリックまたはドラッグにより、選択したの日付セル背景色をハイライト出来る
- 選択した日付セルはその選択数を表示する（n日、n営業日といった表示。n日は選択数、営業日は選択数-土日祝日の数）。選択範囲の開始日、終了日を含む
- 選択数を表示するのはドラッグで連続ドラッグした場合のみ。日付セルのクリックにより飛び飛びの日付を選択することも可能だが、その場合は選択数を表示しない。月をまたいだドラッグの場合も同様に選択数を表示する
- 日付セルのクリックまたはドラッグによるハイライトはクリア可能
- `3ヶ月、1ヶ月、1年ごと` といった表示形式の違いにより、画面表示が変化する。1ヶ月の場合は1列（その月を1列としてカウント）×1行表示、3ヶ月の場合は3列×1行表示、1年の場合は3列×4行表示がデフォルト表示となる。

# 非機能要件

- 対象OS：Windows
- 設定ファイル(`config.toml`)はexeと同フォルダに配置する
- 高速動作するためにRUST製アプリ(egui, eframe, chrono使用)として作成
- `起動からカレンダー表示まで300ms以内`、`表示切替は16ms（1フレーム）以内` を目標として高速で動作することを優先させる
- 初回リリース時は日本語のみ対応、次バージョンでは英語、多言語に対応する
- 祝日は日本の祝日を`koyomi_rs`クレートを用いて計算ベースで判定する。外部APIやネットワーク通信は不要。
- exe形式で起動し、インストールは行わない    
- 終了時、表示位置を覚えておき、起動時は最終表示位置を再現する    
- 最終表示位置が存在しない場合は（初回起動時または最終表示位置が保存されなかった場合）、画面中央に表示する    
- 画面表示形式も記憶しておき、起動時には表示形式を再現する。表示形式とは`3ヶ月、1ヶ月、1年ごと` の表示形式のことである    
- 表示位置・画面サイズ・表示形式の保存について：設定ファイル(`config.toml`)に保存する。保存タイミングは終了時のみ。保存失敗時はエラーログを設定ファイルと同フォルダに出力する(それ以外は何もしない、設定ファイルが存在しない場合はデフォルト設定に従う)。    
- 画面表示形式、画面サイズがそれぞれ記憶されていなかった場合は、標準の表示形式(3ヶ月)・画面サイズを再現する

# PoC
- 見た目をグラスモーフィズムの透明感あるデザインとする。
- 上記デザインを実現する場合、以下の懸念が存在する。
	- eguiは即時モードGUIで、ウィンドウ背景の透過・ぼかしはOS側のComposition API（Windows: `DwmExtendFrameIntoClientArea` やAcrylic/Mica）に依存します。eframeの標準機能では半透明背景はサポートが限定的で、`raw-window-handle` 経由でWin32 APIを直接叩く必要が出てきます。これは「高速動作第一」と矛盾する可能性が高いため、PoC（概念実証）を先に実施する

# 今後実現したい機能・非機能要件（Non-Goals）
- 起動後はショートカットで最小化、(最小化後の)再表示を行うことが出来る
- ピン留め、表示切替（3ヶ月、1ヶ月、1年ごと）、月の切り替え（次月・前月）といった主要な操作もすべてショートカット対応する- 
- 画面サイズも記憶しておき、起動時には画面サイズを可能な限り再現する（ドラッグにより画面サイズを変更可能である）
- ただし、画面に対するドラッグにより画面サイズをある程度変更することは可能である。1年ごとの表示の場合、ドラッグ操作により1列×12列表示も可能となる。1年ごと以外の3ヶ月ごとであっても、1列表示×3ヶ月表示に変更することも可能である。ここでいう1列とは1ヶ月の表示を1列としてカウントしている。
